2. Core Data Structures

How we store collections of data. Choosing the right one is critical for performance.

List `[ ]`

An ordered, mutable (changeable) collection. Your go-to for most sequences of items.

products = ["Laptop", "Mouse", "Keyboard"]
products.append("Webcam")       # Add an item to the end
first_product = products[0]     # Access by index (starts at 0)
print(first_product)            # Output: Laptop

Dictionary `{ }`

A collection of key-value pairs. Essential for working with JSON-like data.

order = {
    "order_id": "ord_123",
    "product_name": "Quantum Laptop",
    "quantity": 1
}
order["status"] = "shipped"     # Add/update a key-value pair
print(order["product_name"])    # Access by key

Tuple `( )`

An ordered, immutable (unchangeable) collection. Use it for data that should not be altered, like coordinates or configuration settings.

# A tuple representing an RGB color
red_color = (255, 0, 0)
# You can access red_color[0], but you cannot do red_color[0] = 100

Set `{ }`

An unordered collection of **unique** elements. Excellent for finding unique items or checking for membership.

# Find unique product categories from a list
categories = ["electronics", "audio", "video", "audio"]
unique_categories = set(categories)
print(unique_categories)  # Output: {'video', 'electronics', 'audio'}

3. Control Flow

How we make decisions and repeat actions.

# if/elif/else
if order["quantity"] > 5:
    print("Bulk order discount applies.")
elif order["quantity"] > 1:
    print("Standard order.")
else:
    print("Invalid quantity.")

# For loop (iterating over a list)
for product in products:
    print(f"Processing product: {product}") # Note the f-string for easy formatting

4. Functions

How we package code for reuse. The core of building modular applications.

# Defines a function that takes two arguments
def calculate_total_price(price, quantity):
    total = price * quantity
    return total

# Calls the function with keyword arguments
final_price = calculate_total_price(price=1200.00, quantity=2)

# A more advanced function with default values and flexible arguments
def process_order(order_id, *items, customer_details={}):
    print(f"Processing order {order_id} for items: {items}")
    if customer_details:
        print(f"Customer: {customer_details.get('name')}")
Guru's Insight: `*args` and `**kwargs`

In function definitions, `*args` collects multiple positional arguments into a tuple. `**kwargs` collects multiple keyword arguments into a dictionary. They allow you to create highly flexible functions that can accept a variable number of inputs.

5. Key Libraries for a Data Architect

You don't need to know everything, but you need to know what tool to reach for.

  • Pandas: The undisputed king for reading, transforming, and analyzing structured data (like CSVs or database tables) in memory.
  • Boto3: The official AWS SDK for Python. How you interact with any AWS service, from S3 to Kinesis.
  • Requests: The standard for making HTTP API calls to pull data from external web services.

6. Error Handling

Production code must anticipate errors. The `try...except` block is how you handle them gracefully.

try:
    # Code that might fail
    price = order["price"] # This will fail, as 'price' is not in our order dict
except KeyError:
    # Code that runs if a KeyError happens
    print("Error: 'price' key not found in the order dictionary.")
except Exception as e:
    # A general catch-all for any other exception
    print(f"An unexpected error occurred: {e}")
