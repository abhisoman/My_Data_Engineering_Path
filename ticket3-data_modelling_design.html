Hands-On Task #3: Senior Engineer / Architect TICKET-003: Data Modeling & Design for Personalization

The Scenario

The business wants to launch a personalization initiative. This requires two things: 1) a very fast way to look up customer profile data to personalize the website in real-time, and 2) a way for the BI team to analyze sales trends based on customer demographics stored in their profiles.

Your Mission

This is a design task. You will not build everything, but you will design the solution and justify your decisions. Create a short design document (a text file is fine) that answers the following:

  1. DynamoDB Table Design:
    • Design the schema for a new `user_profiles` table in DynamoDB. The table must store `user_id`, `email`, `first_name`, `last_name`, `city`, and `signup_date`.
    • **Justify your choice for the Partition Key.** Why is it the right choice for performance and scalability?
    • The marketing team wants to sometimes find users by email address. Propose a solution for this secondary access pattern. What kind of index would you use and why?
  2. Redshift Data Model Design:
    • Design a `dim_customer` dimension table for Amazon Redshift that will store this profile information for analytics.
    • Draw a simple text-based star schema diagram showing how your new `dim_customer` table would join to a `fct_sales` table.
  3. Architectural Flowchart:
    • Draw a simple text-based flowchart showing the AWS services you would use to keep the Redshift `dim_customer` table synchronized with the DynamoDB `user_profiles` table. (Hint: Think about real-time updates).

Acceptance Criteria

  • A design document clearly explaining your choice of DynamoDB keys and indexes.
  • A clear star schema diagram for the Redshift model.
  • A logical flowchart for the DynamoDB-to-Redshift sync pipeline.

Core Concepts Cemented

  • Purpose-built data stores (using the right tool for the job).
  • DynamoDB data modeling (keys, indexes).
  • Redshift data modeling (star schema).
  • High-level pipeline architecture and design.
  • Justifying technical decisions based on business requirements.
