Hands-On Task #1: Junior Data Engineer TICKET-001: Data Landing Verification

The Scenario

The real-time Kinesis pipeline is now active. Before we build the downstream transformation jobs, we need a simple, automated way to verify that data is landing correctly in our S3 data lake every hour. This prevents us from processing incomplete or missing data.

Your Mission

Write a single Python script (`verify_landing.py`) that performs the following checks:

  1. Check for New Data: Using `boto3`, the script must check the `aida-lab-landing-zone` S3 bucket. It should verify that new data files exist within the partition structure created by Firehose for the **current UTC hour**. (Firehose creates prefixes like `YYYY/MM/DD/HH/`).
  2. Check for Metadata: (This step simulates the Glue Crawler's role). Your script does not need to run a crawler, but it should contain a placeholder function and a print statement indicating that "In a real pipeline, we would now trigger a Glue Crawler or check the Glue Data Catalog for the new partition."
  3. Provide a Clear Status: The script should print a clear `SUCCESS: Data found for the current hour.` message if files are present, or a `FAILURE: No data found for the current hour.` message if they are not.

Acceptance Criteria

  • A working Python script that uses `boto3`.
  • The script correctly identifies the S3 prefix for the current UTC hour.
  • The script prints a clear success or failure message to the console.

Core Concepts Cemented

  • Interacting with S3 programmatically using Python (`boto3`).
  • Working with S3 prefixes and object listing.
  • Understanding the relationship between data landing and metadata updates.
  • Basic operational scripting and validation.
